## 1. 本页目标

在 **to C SaaS** 路线下，把「从 MVP 验证 → 可对外售卖、可留存、可证明数据、可多端使用」的工作压成可排序 backlog。  
仍遵守 PRD 已定义的学习闭环：**目标 → AI 计划 → 执行与打卡 → 复盘 → 调整**；SaaS 化增量聚焦 **信任、增长可观测、商业化、成本闭环、移动体验、视频学习环节**。

### 1.1 本路线显式必达范围（全部在排期内完成）

以下与 **RICE 排序独立**：无论分数高低，**M-SaaS-1～2 内须落地可验收版本**（可分阶段灰度）。


| 必达项                 | 验收口径（摘要）                                                                                                                                                                                                            |
| ------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **首启 10 分钟闭环**      | 新用户从进入到完成「首个目标 + 首份 AI 计划 + 当日首个可执行任务」≤10 分钟；可跳过但有关键路径引导                                                                                                                                                            |
| **埋点 + 7 日留存可证**    | 事件字典覆盖激活链路；支持 **D7 留存**（及 D1/D3）分群/ cohort 或等价报表，可回答「首周是否回来」                                                                                                                                                        |
| **信任包**             | 隐私政策、服务条款、账号删除、核心数据导出、AI 数据处理说明；关键入口可见                                                                                                                                                                              |
| **增长 / Onboarding** | 首启引导 + 关键空态/召回文案 + 轻量分享/邀请占位（非复杂社交）；**可观测性**：外呼 LLM 超时导致 **模板回退** 视为可接受降级，**不等同**整站故障（与 `16` §7.1 一致）                                                                                                                                                                                 |
| **轻量商业化 + 配额**      | 订阅/支付 MVP + Free/Pro（或等价档位）+ **AI 与核心增值能力的配额/限流** 可运营                                                                                                                                                               |
| **移动端**             | **Phase 1 + Phase 2 均必达**：Phase 1 为响应式 + PWA + 触控与安全区，手机浏览器可跑通闭环；**Phase 2** 为 **Capacitor 壳 + TestFlight + Google Play Internal** 内测分发（详见 `16-agent-saas-dev-plan.md` Wave 6 / Wave 10）                            |
| **视频学习**            | **必选环节**：**上传与外链两种形态均需支持**，用户/运营可 **自由选择**（同一学习计划或任务下可混用）。外链 **固定白名单**（`bilibili.com`、`youtube.com`、`vimeo.com`，评审可调）；上传 **MinIO 自建** + **分套餐配额**（Free 2GB/单文件 500MB；Pro 20GB/单文件 2GB）；MVP 须含播放、进度、与目标/计划/任务关联及版权提示 |
| **内部运维与可调配置**（支撑必达） | **Ops 管理后台**（独立入口、强鉴权、与 to C 主站隔离）+ **运行配置中心**：**AI 多厂商/代理 Base URL、模型、超时、功能开关**；**API Key / Secret 仅存服务端**（加密或外部 Secret 引用，**禁止**浏览器明文回显）；**Agent 队列健康、失败可读、并发/超时可调**。**当前阶段默认以阿里百炼（DashScope）环境变量为权威**（如 `DASHSCOPE_API_KEY`、可选 `DASHSCOPE_BASE_URL`）；OpenRouter 等为 **可切换 Provider Profile**。验收与波次见 `16` **Wave 3.5**、**§10** |


### 1.2 视频形态补充（产品冻结口径）


| 形态        | 用户价值                        | MVP 须含能力                                                                                    |
| --------- | --------------------------- | ------------------------------------------------------------------------------------------- |
| **外链/嵌入** | 快速接入 B 站、YouTube、Vimeo 等公网课 | URL 校验、iframe/开放平台嵌入策略、**进度可手动或半自动同步**（与任务完成规则在 DoR 写死）                                     |
| **上传**    | 自有版权或内部材料                   | **MinIO**（S3 兼容 API）+ 预签名 URL；分片或大文件策略；MIME/大小限制；**仅登录用户可访问的受控链接**；删除与 **Free/Pro 分档配额** 挂钩 |


**说明**：两种形态并行会增加 **E（人周）** 与 **合规审查面**；不在产品层再「二选一」，由 `16-agent-saas-dev-plan.md` 中的技术波次拆分交付顺序（可先外链后上传或并行双轨，由研发排期决定，但 **验收须两种都可用**）。

## 2. 产品卖点与亮点（SaaS 对外叙事草案）

用于官网/应用商店/投放落地页，与研发 backlog 对齐；具体文案可再经品牌与法务润色。


| 卖点            | 亮点（用户可感知）                | 依赖能力（本页 backlog）  |
| ------------- | ------------------------ | ----------------- |
| **AI 学习教练**   | 从目标到周计划、复盘与周报，少决策、少空想    | AI 计划/复盘/配额、质量门禁、**Provider 可切换与运维可观测**（`16` Wave 3.5）  |
| **闭环可度量**     | 仪表盘与趋势、打卡与成就，看见进步        | 分析页、打卡、埋点         |

| **轻订阅、成本可控**  | 免费档可用，付费解锁 AI/深度能力；不滥用模型 | **支付、套餐、配额**      |

## 3. RICE 计分规则（本页统一）


| 维度               | 取值              | 含义（本页）                                        |
| ---------------- | --------------- | --------------------------------------------- |
| **R** Reach      | 1–5             | 下一季度能触达的 **有效用户面/场景覆盖面**（含已有用户与渠道可放量）         |
| **I** Impact     | 1–5             | 对 **激活率、留存、付费转化、AI 成本可控、合规风险、多端与视频闭环** 的综合影响  |
| **C** Confidence | 50% / 70% / 90% | 对方案可行性与需求确定性的信心                               |
| **E** Effort     | 人周（整数）          | 端到端交付粗估（含前后端、联调、文档、灰度）                        |
| **RICE 分**       | 一位小数            | `(R × I × C) / E`，其中 **C 代入 0.5 / 0.7 / 0.9** |


**排序规则**：RICE 分高者优先；**P0 行内顺序**另考虑依赖（例：支付上线前须有埋点、套餐与配额草案；视频与移动可与主链路并行但须在 M-SaaS-2 前可演示）。**标记「必达」的不因分低而取消。**

## 5. 必达项与 RICE 的并行关系（避免误解）

- **必达清单（§1.1）** 保证「该做的都进排期」；**RICE** 用于 **同一 Sprint 内** 的取舍与人力分配。  
- **视频学习**、**移动端 Phase 1** 分数可能偏低（E 大），仍须在 **M-SaaS-2 结束前有可演示 MVP**，可与商业化并行不同成员。  
- **首启 10 分钟** 与 **埋点 D7** 强相关：Onboarding 步骤须带 **步骤级事件名**，便于证明「完成首启漏斗的用户」的 D7。
- **运维后台 / 配置中心（§1.1 支撑必达）** 可与 **视频 Wave 4** 并行人手，但若 **无主备运维界面**，AI/Agent **排障成本**将阻塞增长与视频验收；默认排期：**不晚于 Wave 4 结束前** 完成「可演示」最小集（见 `16` Wave 3.5）。

## 7. 显式 Out of Scope（本阶段仍不做）

- 企业多租户 LMS、复杂权限树（与 PRD **非目标用户** 一致）  
- **强社交**学习社区、UGC 广场、小组课运营（与 PRD 一致；**轻分享链接保留**）  
- 自制 **长视频平台级** CDN/版权交易（**不排除**对接第三方视频托管与外链）  
- **纯原生** App 从零重写（与 **Capacitor + 现有 React** 路线冲突，不做）  
- **Capacitor 已纳入必达**（见 §1.1、§4）；本节不再将「壳应用」列为 Out of Scope

## 8. 评审检查清单（DoR 摘要）

- **视频双形态**：外链白名单（`bilibili.com` / `youtube.com` / `vimeo.com`）与 **MinIO** 凭据、**Free 总 2GB/单文件 500MB、Pro 总 20GB/单文件 2GB** 配额、**版权与用户责任条款** 已拍板并公示  
- **移动端 Phase 1** 设备矩阵与「最小可接受」断点已评审  
- **Apple / Google 开发者账号** 已申请或流程已启动；签名与内测渠道配置就绪（Capacitor 必达）  
- **微信 / 支付宝商户号**、营业执照、对公、**ICP 备案** 路径已明确（生产前）；Stripe 区域与币种已拍板  
- Free/Pro 权益表与 **AI/视频存储配额** 已签字  
- **MinIO** bucket 命名、访问策略、**备份**（如 `mc mirror` 到冷备）责任人与周期已登记  
- 埋点事件字典与 **PRD §8** 及 **D7 cohort** 定义一一映射  
- 隐私/删除/导出/AI 说明文案经审阅  
- 灰度与回滚遵守 **12-release-rollout-and-rollback-runbook**  
- **卖点 §2** 与落地页、应用商店描述一致（无过度承诺）
- **AI 默认厂商与密钥**：产品与研发对齐 **`16` §8.1 Q7**：当前阶段 **阿里百炼（DashScope）** 为主；`DASHSCOPE_API_KEY`（及可选 `DASHSCOPE_BASE_URL`）**不得在仓库明文**；**Ops 后台**仅展示密钥状态与轮换入口，不写死单一厂商到前端 bundle
- **Agent / AI 排障**：任务失败须在后台或数据库字段可见 **可读错误**（对标 `16` §7.1：`error: {}` 序列化补强）
